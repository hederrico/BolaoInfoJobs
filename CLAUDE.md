# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

**BolГЈo 015** вҖ” a single-page web app (UI in Brazilian Portuguese) for a World Cup 2026 prediction pool. A fixed group of friends predicts scorelines; the app shows live scores, scores each prediction, and ranks players in real time. (Brand string is "BolГЈo 015"; the internal localStorage key is still `bolao_meli_live_v1` for back-compat вҖ” don't rename it.)

The entire app is one file: [index.html](index.html) вҖ” HTML + CSS in `<style>` + vanilla JS in `<script>`. There is **no build step, no framework, no package manager, no tests, no lint config.** Dependencies are loaded from CDNs at runtime (Supabase JS, Google Fonts, flag images).

## Running / deploying

- **Run:** open `index.html` in a browser, or serve the folder statically (e.g. `python -m http.server`). A network connection is required вҖ” the app fetches the live feed, Supabase, fonts, and flag images from external services.
- **Deploy:** commit and push to `origin` (GitHub: `hederrico/BolaoInfoJobs`). There is no CI/build; the repo is the deployable artifact.

## Architecture (the non-obvious parts)

### Three layers of data, with fallback
1. **Live match feed** вҖ” `GET https://worldcup26.ir/get/games` (public, no auth), polled every 30s via `tick()` вҶ’ `fetchGames()`. Populates the global `GAMES` array. The feed carries the **whole tournament** (104 games). Key per-game fields: `type` (`group` | `r32` | `r16` | `qf` | `sf` | `final` | `third`), `group` (`A`вҖ“`L` for group games), `matchday`, `home_team_id`/`away_team_id` (numeric string; **`'0'` = team not yet decided** in knockout), `home_team_label`/`away_team_label` (knockout source, e.g. "Winner Match 74" / "Winner Group E" / "3rd Group A/B/C/D/F"), `home_score`/`away_score`, `finished` (`"TRUE"`), `time_elapsed`, `local_date`, `stadium_id`. The `/get/stadiums` endpoint gives each `stadium_id`'s city/country/region.
2. **Shared predictions** вҖ” Supabase table `palpites` (`game_id, player, home, away`, upsert `onConflict: "game_id,player"`). `supaPull()` reads all rows into `GUESS`; `supaUpsert()` writes on each keystroke. URL + anon key are hardcoded near the top of the script.
3. **Local fallback** вҖ” if Supabase is unavailable, predictions persist to `localStorage` under key `bolao_meli_live_v1` (`saveG`/`loadG`), and if even that fails, to the in-memory object `mem`.

`GUESS` is the single source of truth for predictions, shaped as `{ gameId: { player: {a, b} } }` where `a`/`b` are home/away scores as strings (`""` = no prediction).

**Stream links** (for the "Ao vivo" tab) live in a parallel structure `STREAMS` (`{ gameId: "youtubeId" }`) with the **same 3-layer fallback**: Supabase table `match_streams` (`game_id, youtube_id`; `streamsPull()`/`streamUpsert()`) вҶ’ `localStorage` key `bolao_streams_v1` (`loadStreams`/`saveStreams`) вҶ’ `mem`. `streamUpsert()` silently no-ops if the table doesn't exist yet, so the app works front-only and "promotes" to shared once the table is created (read public + anon write, mirroring `palpites`). `ytId()` extracts the 11-char id from any YouTube link (watch / embed / youtu.be / live / shorts) or accepts a raw id.

### Scoring model (`pts()`)
- **Cravou** (exact score): +6 В· **Vencedor/empate** (correct outcome): +3 В· **Errou**: 0 В· no result yet: `pend`.
- `BASELINE` holds each player's points accumulated **before** this app existed; `buildGeral()` sums `BASELINE + finished games + live games` to produce the live standings. `PLAYERS` is the fixed roster вҖ” both are hardcoded constants.

### Tabbed views & the standings/scenarios engine
The app is split into six tabs (`activeTab`, built by `buildTabs()`, dispatched by `renderActive()`):
- **BolГЈo** (`render()`) вҖ” the original prediction/scoring view (live cards, prГіximos, geral, histГіrico). Live group game cards also embed a live mini-standings table. When no game is live, the "Ao vivo" slot shows a **countdown card** (`buildNextCountdown()`/`fmtCountdown`) to the next upcoming game (flags + names + kickoff + a 1s-ticking timer; `_cdTimer`, cleared by `stopCountdown()` on re-render/tab-switch; fires one `tick()` when it hits zero).
- **Assistir jogos** (`renderLive()`; tab id is still `"live"`) вҖ” a **CazГ©TV stream mosaic** (**max 2 games**, "switcher com foco" model) that fills the **maximum available viewport space** (`sizeMosaic()` sets the `.mosaic` pixel height = viewport вҲ’ its top, recomputed on resize/orientation; **1 game = huge**, **2 = split** вҖ” stacked in portrait, side-by-side in landscape via `.mosaic.mos-1/.mos-2` grid rows/cols). The `.live-selector` strip lists **live + upcoming-with-stream + finished-with-stream** (so past games can be re-watched), recent-first; live games auto-select, otherwise the user picks. `.msq` is a flex column (header + `flex:1` stage), no fixed aspect-ratio, so the video scales to the cell; each `.msq` has a **Г— remove** button (`liveToggle`) to drop it. **Layout toggle** (`liveLayout` null=auto/orientation | `"stack"` | `"side"`, button in `.lh-acts`, only with 2 games): `toggleLayout()` swaps the `.mosaic.lay-*` class **without rebuilding the players** (just re-grids + `sizeMosaic()`). **Watch-mode** (`body.watch-mode`, toggled in `renderActive()` when `activeTab==="live"`): hides `.top` (brand+legend) and `.statusbar`, and collapses `.tabs` into one horizontal-scroll row вҖ” so the videos get the maximum space (`sizeMosaic` measures the reduced top and grows the mosaic). Uses the **YouTube IFrame Player API** (lazy-loaded once via `loadYT()`; not raw iframes) so `mute/unMute/playVideo/pauseVideo` are scriptable. **Android** plays both; tap a quadrant (`focusLive()`) only moves the audio. **iOS** (`IS_IOS()`/`liveIOS()`, overridable via `_liveForceIOS` for tests) plays **one at a time** вҖ” non-focused quadrants `pauseVideo` (iPhone won't run two videos at once). Players start `mute:1 + autoplay:1 + playsinline:1` (else iOS forces fullscreen / blocks autoplay). The `youtube_id` per game comes from `STREAMS` (see data layers). **Stream link sources** (in priority): (a) manual вҖ” a **password-gated** manage mode (`liveManage`, same `ADMIN_PW` as history edit, toggled by the `.livelock`) shows a paste field per game (`ytId()` parses it); (b) **auto-fetch via the YouTube Data API** (see `## YouTube Data API` below) вҖ” `autoFetchLive()` runs in `tick()` and, **only if** a live game is missing a link, does **one** `search.list` (covers all live games), throttled to 3 min; (c) a **backfill button** in manage mode (`backfillStreams()`) for past + scheduled. Manual always wins (it overwrites). The mosaic auto-selects up to 2 live games that have a stream; a horizontal `.live-selector` strip swaps which 2 show (`liveToggle`, drops oldest past 2). Layout: portrait stacked, landscape side-by-side (`.mosaic.mos-2`). Each quadrant has an "abrir no YouTube" deep-link fallback in case a future video disables embedding (CazГ©TV currently publishes with `playableInEmbed: true`, verified). **Guard:** `renderLive()` is keyed by `_liveSig` (selected ids + their stream ids + iOS/Android + manage) and only rebuilds the players when that changes вҖ” otherwise `refreshLiveChrome()` updates just the header/selector/scores so the 30s `tick()вҶ’renderActive()` doesn't reload running videos; `stopLive()` destroys players on tab-switch.
- **Grupos** (`renderGroups()`) вҖ” standings tables for all 12 groups (recomputed live) plus a **best-thirds** ranking (`thirdsRanking()`: the 12 third-placed teams ordered pointsвҶ’GDвҶ’GF, top 8 qualify вҖ” FIFA Art. 13 criteria 1вҖ“3; the further tiebreakers fair-play and FIFA-ranking aren't in the feed, so a stable group-letter order resolves remaining ties; a note in the thirds card states this). The qualifying thirds get a "Classificando" badge in their group table (`qualThirdSet()`).
- **Mata-mata** (`renderBracket()`) вҖ” vertical knockout split into the two halves of the draw ("lado de cima"/"lado de baixo"). The bracket tree is reconstructed from the feed: each knockout game's `home_team_label`/`away_team_label` names its source ("Winner Match 74", "Winner Group E", "3rd Group A/B/C/D/F", "Loser Match 101"), and the game `id` IS the official FIFA match number (group 1вҖ“72, r32 73вҖ“88, r16 89вҖ“96, qf 97вҖ“100, sf 101вҖ“102, third 103, final 104). `koTree()` walks the source labels from the two SFs (M101/M102) to compute each half's match set and a DFS order that keeps bracket siblings adjacent. The view is a **horizontal connected bracket**: round columns (16-avosвҶ’Final) laid out with `justify-content:space-around`, and an absolutely-positioned SVG overlay (`drawBracketLines()`) draws the elbow connectors by measuring each card's real position (`getBoundingClientRect`) and linking each match to its two source matches вҖ” robust to variable card heights. Each card shows its number + both sources (`ptKoLabel()` translates them), so undecided slots still show who-plays-whom (e.g., "Venc. Jogo 74"). Horizontal scroll on `.bkscroll`; 3rd-place sits below the tree.
- **CenГЎrios** (`renderScenarios()`) вҖ” per-group qualification scenarios. The same compact scenario is also embedded (collapsible) inside live/upcoming game cards on the BolГЈo tab via `cardScenarioSection()`.
- **Corrida** (`renderRace()`) вҖ” a **bar-chart race**, one frame **per goal**. `parseScorers()` reads the feed's `home_scorers`/`away_scorers` (Postgres-array strings like `{"Name 27'","Name 45+2'"}` вҶ’ `{side,name,min}`, stoppage `45+2`вҶ’`45.02` for ordering). `raceFrames()` collects all goal events of finished/live games **that have a palpite** (`hasAnyGuess`), orders them by absolute time (`kickParts().ts + min*60000`), and walks them with an **incremental** per-game-points update so cumulative = `BASELINE + ОЈ_games pts(score-so-far)` вҖ” the **last frame exactly equals `buildGeral`'s live totals** (verified). Frame 0 = `BASELINE` ("Largada"); the first goal frame is the bolГЈo's first palpite. Each frame carries the scoring match (`hid/aid/h/a/min/scorer`) shown in the header with **both flags** + score (scoring side highlighted) + `min' В· scorer`. Bars are 10 absolutely-positioned `.racerow`s reordering via `translateY(rank*ROWH)` + `.rr-fill` width (FLIP, no lib); numbers tween via `requestAnimationFrame`. Bars use each player's fixed color + avatar; below the race sits the all-players **evolution line chart** (`lineChartSVG`) with a clickable legend. Controls: play/pause (`.rc-play`) + scrub `.rc-slider`; step duration adapts to frame count. **Guard:** `renderRace()` early-returns when `_raceSig` (finished ids + live ids/scores) is unchanged and a `.racewrap` exists, so the 30s `tick()вҶ’renderActive()` doesn't restart a running animation; `stopRace()` clears timer/RAF when leaving the tab.

Icons are inline SVG (`ICONS`/`icon()`), not emoji. The scenario grid is `table-layout:fixed` so it fits mobile width; outcome cells show only a flag (winner) or `=` (draw). The current scenario (matching the live/now scores) gets a gold outline on its whole `<tr>` (`s.cur`, computed in `scenarioFor`).

### Player identity, profiles & charts
- **Avatares abstratos + cores por jogador** вҖ” `PLAYER_META` fixes each player a soft primary `base` (+ `light`/`accent` secondaries) and a `motif` (geometric composition: diag/horizon/quarters/stripes/concentric/zigzag/overlap/diamond/arcs/bars). `avatarInner(p)` builds the clipped-circle SVG (base fill + `AV_MOTIFS[motif]`); `avatarHTML(p,size)` wraps it for the DOM. `pColor(p)`=base (ranking/profile names); `pSoft(p)`=`mix(base,#8693A4,.34)` вҖ” a **neutral/opaque tone** used for the Corrida bars and line charts (less aggressive). Avatars appear **everywhere a player shows**: Geral ranking, palpite lists (`.urow`/`.guess` `.who`), profile, Corrida bars + legend, and the share canvases. **Canvas avatars** reuse the exact SVG: `avatarSVGRaw(p)` вҶ’ `loadAvatar(p)` rasterizes a `data:image/svg+xml` (URI-encoded, no taint вҖ” verified) into an `Image`; `drawAvatar()` clips it to a circle. `shade()`/`mix()` are the color helpers.
- **Perfil do jogador** (`openProfile(p)`) вҖ” modal opened by clicking any ranking row, race row, or chart legend chip. `playerStats(p)` computes (over finished games **with a palpite**) cravadas/acertos/erros, aproveitamento (`pts/(nPalp*6)`), current & record streak, and top teams by points (`byTeam`). The body: stat grid вҶ’ single-player **evolution line chart** (focus-mode, with the bolГЈo **average dashed line**) вҶ’ **VocГӘ vs. o bolГЈo** (your total vs the field average + delta) вҶ’ **DNA do palpiteiro** (`palpiteDNA(p)`: placar favorito, gols/palpite, mandante/empate/visitante tendency bar, an "estilo" tag вҖ” computed over *all* the player's palpites) вҶ’ **Momentos da campanha** (`playerMoments(p)`/`momentHTML`: "melhor sacada" = the game where the player most out-scored the field, "maior vacilo" = a 0-pt game where most others scored, each with flags + scoreline) вҶ’ top-teams-with-flags. (The old best/worst-**round** box was removed вҖ” it read identically when only one round had palpites.) The header has a corner **share** button вҶ’ `makeProfileCanvas(p)` (a full-profile share image: header, stat grid, evolution chart rasterized via `svgToImage`, vs-bolГЈo, DNA, momentos, top teams); it calls `closeProfile()` first so the share modal isn't stuck behind the profile overlay. `playerMoments` counts scored/cravaram over the **whole roster of 10** (`tot`), not only those who palpitated that game.
- **`lineChartSVG(frames, players, opts)`** вҖ” pure-SVG multi-line chart (no lib); x = per-goal frame index, y = cumulative points. `opts.soft` uses `pSoft` (neutral tone); `opts.focus` highlights one player (others faint); `opts.upTo` reveals only frames `0..upTo` (bright) over a faint full preview + a playhead line; `opts.avg` adds a dashed **mГ©dia do bolГЈo** line + label. Element sizes are parameterized вҖ” `opts.thick` (line width), `opts.endD` (end-avatar size), `opts.endFs` (end-name font), `opts.axisFs` (axis numbers) вҖ” so the chart can render large. Each shown line **ends in the player's avatar (nested `<svg>` reusing `avatarInner`) + name**, with **vertical de-collision** so labels never overlap (a thin connector links each back to the line's true endpoint); disable with `opts.endlabels:false`. `lineChartSVG` is now used by the **Profile** (focus-mode) and the **Profile share image** (rasterized via `svgToImage` вҖ” a data-URI SVGвҶ’`Image`, no taint). Shares `raceFrames()` data.
- **Corrida evolution chart = animated `<canvas>`** (`drawChartCanvas(R)` inside `renderRace`, full-width + same height as the race bars `PLAYERS.length*ROWH`). `R` is a **fractional** frame index: the avatar+name end-markers (and playhead + revealed line tips) **glide smoothly** between positions instead of jumping per goal. `applyFrame`/`tweenVals` interpolate `chartR` from the previous to the target frame over `DUR` inside the same rAF loop that tweens the bar numbers, so the chart, bars, and numbers animate in lockstep (and scrub together). Avatars are rasterized once via `loadAvatars` (`_AV`); width is cached in `_lcW` (`measureLc()`) to avoid per-frame layout reads.
- **Video export** (`recordRaceVideo()`) вҖ” re-renders the race + evolution **frame-by-frame onto an offscreen canvas** (`drawBars`/`drawChart`, interpolating both values and rank positions between goal frames; the chart draws each player's **avatar + name** end-markers with de-collision, like the live chart) and records via `canvas.captureStream(30)` + `MediaRecorder`. The mime list **prefers MP4/H.264** (`video/mp4;codecs=avc1вҖҰ`, WhatsApp-compatible) and only falls back to webm when unsupported (the result subtitle warns when it's webm). Progress, looping `<video>` preview, and actions live in the `#vidov` overlay (`closeVideoOverlay`/`shareVideo`/`downloadBlobAs`; transient state in `_vid`). **Download is always shown; "Compartilhar" only on mobile** (`pointer:coarse`/mobile UA) вҖ” desktop video-share via the OS sheet is unreliable, so desktop is download-only. The share `File` uses a **clean MIME** (`_vidType()` вҶ’ `video/mp4`/`video/webm`, stripping the `;codecs=вҖҰ` that the blob carries, which broke the native share), and `shareVideo` falls back to download if `navigator.share` rejects. Triggered by the `.rc-vidbtn` "Exportar vГӯdeo" icon in the race header corner.

### Engagement features (all reuse the scenario engine)
- **Status selo** (`teamStatus()`/`statusBadge()`) вҖ” per team in group tables: green вң“ *Classificado* (guaranteed top-2 = `allPosвҠҶ{1,2}`), red вң— *Eliminado* (`allPos={4}` / finished 4th), gold вң“ *Classificando* (provisional best-third). Best-third status is intentionally "provisional" while other groups are unfinished (cross-group, not guaranteed).
- **"O que estГЎ em jogo"** (`stakesFor()`/`stakeChipsHTML()`) вҖ” for each of the two teams, the possible finishing positions if THIS game ends win/draw/loss (marginalizes the other remaining games via full scoreline enumeration). Cached in `stkCache` by `groupSig`. On **live cards** the three `V`/`E`/`P` chips render **directly under each team's name inside the scoreline** (`.team .stkmini`, color-coded ok/mid/out via `posClass`); on **upcoming cards** the legacy one-line-per-team block (`buildStakes()`) stays in the collapsed body.
- **Share / print** вҖ” canvas builders, all CORS-safe via **`crossOrigin="anonymous"` flags** (flagcdn.com sends `Access-Control-Allow-Origin:*`, so drawing flags does **not** taint the canvas вҖ” verified). `loadFlags(ids)` preloads (cached in `_flagImg`) and returns a `idвҶ’Image` getter; `drawFlag()` clips a rounded cover-fit flag; shared chrome via `cvBase`/`cvBrand`/`cvFooter`/`cvLegend`. Builders (all `async` except the flag-free ranking): `makeShareCanvas()` (ranking, statusbar + Geral panel вҖ” uses `rankRows()` which now mirrors `buildGeral`: **tie-aware positions** via `compRanks` (equal points вҶ’ same rank, e.g. two #4 then #6) and a **movement arrow** per row, `mv = basePos(before live) вҲ’ rank(now)`, drawn only when games are live, summing all parallel live games), `makeMatchCanvas(g)` (one match: flags, **white** score, each player's prediction + points colored to match the DOM panel via `PTCOL` вҖ” cravou green / vencedor gold / errou gray, **no action-word text** вҖ” plus the live group mini-standings "como estГЎ o grupo"), `makeGroupCanvas(L)` (full group table + status dots, Grupos tab), `makeThirdsCanvas()` (best thirds, Grupos tab), `makeScenarioCanvas(L)` (CenГЎrios tab вҖ” `scenarioGridCanvas`/`scenarioAggCanvas`/final standing), `makeBracketCanvas()` (Mata-mata tab вҖ” full knockout from `koTree()`: 5 round columns with bracket-style y-centering (`yc[id]` = avg of its two sources' centers), elbow connector lines, per-match `drawKoCard()` showing match number + both teams/flags or translated source labels (`ptKoLabel`) + scores, plus 3rd-place; `cvTrunc()` ellipsizes long labels). `makeBracketCanvas()` lays out the two halves with a `halfGap` separator and measures-free fixed `pitch`/`cardH` so cards never overlap; the title sits clear of the column-header row. All go through `openShareModal(builder, filename, title)` where **builder may be a canvas or a (sync/async) function returning one**; it shows the `#shmodal` preview (`max-height:90vh`, scrollable, spinner while building) so the result is always visible, then offers **Baixar imagem** (`doDownloadBlob`, always) and **Compartilhar** (`doShareBlob` вҶ’ `navigator.share({files})`, only when `navigator.canShare` supports files). Sharing is **icon-only**: a small `.cardshare` button in the **top-right corner of each panel** (live card header, Geral panel `.gershare`, each history row `.rowshare`, every group + thirds card, every scenario card, and the Mata-mata `sectitle`). There is **no global "Compartilhar" button in the top status bar** (removed).
- **Pull-to-refresh** вҖ” touch handlers (bottom IIFE) drive the `#ptr` indicator and call `tick(false)`.

The shared engine: `tableFromResults()` builds a table from a result list; `orderGroup()` sorts by **points вҶ’ head-to-head mini-league вҶ’ overall GD вҶ’ overall GF** (note: this is H2H-before-GD, matching the reference scenario tool, *not* strict FIFA order; fair-play/drawing-of-lots aren't modeled). `scenarioFor()` produces, for a group on its final matchday (вүӨ2 games left), the reference-style grid: for each outcome combination it **enumerates scorelines** (margins 0вҖ“6) and unions the finishing positions, so a position cell shows multiple flags when the order genuinely depends on goal margins. Results are memoized in `scnCache` keyed by `groupSig()`. The scenario logic was validated to reproduce the official "Group L scenarios" grid exactly.

### Render vs. dynamic-update split (important when editing UI)
- `render()` rebuilds the whole DOM (sections: Ao vivo / PrГіximos / Geral / HistГіrico). The **HistГіrico is grouped by day** (BrasГӯlia): `kickDayInfo(g)` gives a `{key,label}` (weekday В· dd/mm), games are bucketed per day and each bucket gets a `.dayhead` (label + match count + divider) before its cards. A **password-gated edit mode** (`editMode`, toggled by the `.histlock` lock icon on the HistГіrico title via `toggleEdit()` вҶ’ `prompt()` vs the front-side `ADMIN_PW` constant вҖ” a soft gate, not real security) lets you **populate predictions retroactively**: while on, the history lists **all** finished games (not just `hasAnyGuess` ones), empty ones get a gold `.needfill` highlight + "sem palpites", and expanding a card lazily mounts an editable `guessGrid(g.id)` whose keystrokes go through `setGuess` вҶ’ `supaUpsert` (shared) + `saveG` (local). Edit cards don't register `histRefs`, so `refreshDynamic()` leaves them alone.
- `refreshDynamic()` updates scores, clocks, and rankings **in place without recreating inputs**.
- `tick()` chooses between them using a signature string (`sig()`): full `render()` only when the set of live/upcoming games changes; otherwise `refreshDynamic()`.
- **`inputFocused()` guard:** never rebuild while a user is typing a prediction вҖ” it would lose focus/input. Several handlers (`onblur`, `refreshDynamic`) check this. Preserve this invariant when changing the input grid.

### Match classification (`classify()`)
Each game is `finished` (feed `finished === "TRUE"`), `live` (`time_elapsed !== "notstarted"`), or `upcoming`. This drives which card builder runs: `buildLiveCard` / `buildSoonCard` / `buildHistoryCard`.

### Timezone conversion (`kickParts()`)
The feed's `local_date` is in **each stadium's local timezone**, and WC2026 spans four offsets (Mexico вҲ’6; US Central вҲ’5; US/Canada Eastern вҲ’4; Western вҲ’7). `STADIUM_OFFSET` maps `stadium_id` (1вҖ“16, from the `/get/stadiums` endpoint) вҶ’ UTC offset; `kickParts()` converts stadium-local вҶ’ UTC вҶ’ BrasГӯlia (`BRT_OFFSET` вҲ’3). A single fixed offset is wrong (it makes Central/Pacific games an hour or more off). If kickoff times look wrong, check the stadium's offset in the map.

### Teams & flags
`TEAMS` maps the feed's numeric `team_id` вҶ’ `{ pt: "<name>", code: "<ISO>" }`; `flag()` builds a flagcdn.com URL from the ISO code. Adding/correcting a team means editing the `TEAMS` map.

## YouTube Data API (auto-fetch of CazГ©TV links)

The "Ao vivo" tab auto-discovers the CazГ©TV broadcast for each game. Config near the storage block: `YT_API_KEY` (HTTP-referrer-restricted to `hederrico.github.io`; `""` disables auto-fetch) + `CAZE_CHANNEL_ID` (`UCZiYbVptd3PVPf4f6eR6UaQ`). All searches are `search.list` filtered by `channelId` (100 units each; 10k/day quota).

- **Matching** (`ytTitleGame`/`ytTeamIn`): the video title is tokenized into a **word set** (accent/punctuation-stripped via `ytNorm`); a team matches if **all** its significant tokens (len вүҘ 4, stopwords dropped) are present **as whole words** вҖ” so "IrГЈ" (`ira`) does NOT match inside "Iraque", and `_YT_ALIAS` covers CazГ©TV spellings ("TchГ©quia"вҶ’id 4, "EUA"вҶ’id 13). **Critical guard:** a team with no tokens (undecided knockout slot, `team_id 0`) returns `false` вҖ” otherwise `[].every()===true` would vacuously match every video (this bug silently mis-seeded all knockout games once; keep the guard).
- **Live** (`autoFetchLive()`, called from `tick()`): runs **only if** a live game lacks a link, then does **one** `eventType=live` search covering all live games, throttled 3 min (`_liveFetchAt`). One call fills every live game.
- **Backfill** (`backfillStreams()`, button in manage mode): past via `q="JOGO COMPLETO"` (the phrase is **required** in the title, so highlight reels are skipped) + scheduled via `eventType=upcoming`, both paginated/capped.
- **Key exposure caveat:** the referrer restriction stops casual browser theft but **is spoofable** server-side (a `curl` with `Referer: https://hederrico.github.io/` works). Worst case is quota drain (auto-fetch pauses for the day; manual still works) вҖ” no billing is enabled, so no cost. Accepted for a front-only app; a server proxy would be the only real fix.
- **Populating the past:** the `match_streams` table + a one-time seed (66/66 finished games + scheduled-with-defined-teams) were **already applied** to Supabase. The one-off `supabase/*.sql` + `gen-seed.mjs` artifacts were **removed from the repo after applying** вҖ” to repopulate or refresh, just use the **Buscar no YouTube** button in manage mode (`backfillStreams()` runs the identical matcher in-app). The seed was matched by the same `ytTeamIn` logic; the only manual fixups are CazГ©TV spelling aliases (already in `_YT_ALIAS`).
- **Watch buttons & floating player:** every game reference on the **BolГЈo** tab (live cards, prГіximos, and the whole histГіrico) shows a green **play** button (`watchLink()` вҶ’ `<button class="watchbtn" data-gid>`; one delegated document click handler) when `STREAMS[gid]` exists. PrГіximos shows up to **4** games. Clicking it opens `chooseWatch(gid)` вҖ” a sheet with two options: **Assistir aqui** вҶ’ `openFloat(gid)` (a draggable **floating PiP** appended to `<body>`, so it persists across tab switches; `fpDrag()` via pointer events; `closeFloat()`; the `fp-tab` button moves it into the "Assistir jogos" tab) or **Abrir na CazГ©TV** вҶ’ YouTube in a new tab. (Real `requestPictureInPicture()` can't be called on a cross-origin YouTube iframe, hence the custom floating div.) **Embed origin gotcha:** the YouTube IFrame player refuses to play from a `file://` origin (error 150/153) вҖ” it needs an http(s) origin. So in-site playback only works **served** (GitHub Pages https, or `python -m http.server` вҶ’ `http://localhost`), not by double-clicking `index.html`. Players pass `playerVars.origin = location.origin` (when http), and an `onError` handler (`ytFailHTML()`) replaces the failed embed with a friendly "Assistir no YouTube" fallback button instead of YouTube's raw error.
